# Persistent Sales Assistant Agent (Backend + AI Agent)

This repo implements the take-home: a hosted conversational API with **persistent cross-session memory**, **real tool use**, and a **structured self-eval block on every response**.

## Live URL (Railway)

After deploying to Railway, paste your URL here:

```
https://YOUR-RAILWAY-APP.up.railway.app
```

## Endpoints

- `POST /chat/{user_id}` → send a message, get a response + self-eval + tools_called + session_id
- `GET /chat/{user_id}/history` → full conversation history across all past sessions
- `DELETE /chat/{user_id}/memory` → wipe a user's memory (GDPR-style reset)
- `GET /chat/{user_id}/evals` → **bonus**: aggregated eval scores across sessions for a user
- `GET /catalog` → returns the product/pricing catalog the agent uses
- `GET /health` → service health check

## Memory design decision

- **Goal:** true cross-session persistence without re-sending prior context in the request body.
- **Implementation:** store `chat_messages` (history) + a tiny `user_memory_kv` table for extracted facts like:
  - `last_plan_discussed = "Enterprise"`
  - `interests = "SSO, audit logs"`
- **Abstraction:** the agent uses `MemoryStore` (interface) from `app/memory/base.py`. Swapping SQLite → Postgres/Mem0 requires changing only `app/memory/sql_memory.py` (and the DB URL).
- **At scale:** replace KV + raw history with:
  - summarization (rolling summaries) + vector retrieval over conversation chunks
  - a dedicated memory store (Redis + Postgres, or a managed memory product like Mem0)

## Tool use (real callable functions)

Tools are **real Python functions** (not string-injected):

- `search_catalog(query)` → searches `catalog.json` (keyword-ish with fuzzy scoring)
- `get_user_memory(user_id)` → queries the DB via the memory abstraction
- `(bonus)` `flag_for_human(user_id, reason)` → stubbed as a structured payload (hook for queue/CRM)

## Eval design (self-scoring on every response)

Every `/chat` response includes:

```json
{
  "response": "…",
  "eval": {
    "groundedness": 0.91,
    "relevance": 0.88,
    "confidence": 0.85,
    "flagged": false,
    "reasoning": "…"
  },
  "tools_called": ["search_catalog", "get_user_memory"],
  "session_id": "uuid"
}
```

- If `OPENAI_API_KEY` is set, the eval block is generated by an LLM in **strict JSON mode**.
- If not, a deterministic heuristic eval runs so the project is always runnable for demos/interviews.
- All evals are **logged to DB** in `eval_records` and can be aggregated via `GET /chat/{user_id}/evals`.

## Quickstart (local)

```bash
cd persistent-sales-assistant
python -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt --break-system-packages

# Optional (better responses + eval): set OpenAI key
export OPENAI_API_KEY="..."

uvicorn app.main:app --reload
```

Open:
- API docs: `http://localhost:8000/docs`
- Demo UI: `http://localhost:8000/`

## Two curl commands (cross-session memory demo)

> These two calls demonstrate persistence across separate requests using only `user_id`.

**Call 1: establishes plan context**

```bash
curl -s -X POST "http://localhost:8000/chat/demo-user" \
  -H "Content-Type: application/json" \
  -d '{"message":"What is your enterprise pricing?"}' | jq
```

**Call 2: follow-up uses stored memory (no re-sent context)**

```bash
curl -s -X POST "http://localhost:8000/chat/demo-user" \
  -H "Content-Type: application/json" \
  -d '{"message":"Does that include SSO?"}' | jq
```
